QtStereoscopy

A Qt module that adds support for a multitude of stereoscopic display devices.

Requirements and Constraints

QtStereoscopy extends functionality added to the revised QtGui module introduced in Qt5 meaning prior versions of the Qt library are not supported. While the new QtGui module provides a modern way of using the OpenGL API, it is still relatively new (as of September 2014) and a number of features that exist in prior versions of Qt are still lacking. One such feature is a set of public API calls that allow for direct access to the Operating System's native display device handle (required by the Oculus SDK, for example).

The current workaround is to use a part of the Qt Platform Abstraction (QPA) which is currently strongly-coupled to the version of Qt it is built against. Since the QPA API may change in future versions of Qt, this is not a very portable solution but will very likely become one as QPA matures.

Installing QtStereoscopy

QtStereoscopy's build process is still not tied to Qt which means the usual way a Qt module is integrated into a project will not work. Instead, it relies on a set of project include files (*.pri), each of which specifies the build configuration for a specific set of stereoscopic display devices.

Start by cloning the qtstereoscopy project then include one or more project include files (found in the install directory) to a project file (*.pro). For example, to add support for Oculus VR devices, prepend include(</path/to/qtstereoscopy>/install/oculusvr.pri) where </path/to/qtstereoscopy> is qtstereoscopy's location.
